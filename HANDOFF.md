# HANDOFF

## 1. このアプリの説明

報連相アシスタントは、会社員が上司や先輩へ進捗報告・相談をする前に、タスク名、締切日、完了した作業、残っている作業、予定時間、困りごと、メモを入力すると、Teams や Slack に貼り付けやすい箇条書きの送信用メッセージを作る静的Webアプリです。残作業ごとに開始予定日・開始予定時・終了予定日・終了予定時を入れると、9:00〜18:00 の稼働時間だけを使って残工数を自動計算し、土日祝日を除いた締切までの営業日とあわせて、遅延リスクや相談事項を整理します。

## 2. ローカルでの実行方法

依存関係はありません。ローカルサーバーで確認する場合は、このフォルダで以下を実行します。

```bash
npm run dev
```

表示された `http://localhost:5173` をブラウザで開いてください。本番用の静的ファイルを作る場合は以下を実行します。

```bash
npm run build
```

生成された `dist/index.html` を確認する場合は以下を実行します。

```bash
npm run preview
```

## 3. ファイル構造

- `BRIEF.md`: 最初の仕様メモです。対象ユーザー、解決したい問題、UI要件、技術制約が書かれています。
- `index.html`: アプリ本体です。HTML、CSS、JavaScript がすべてこの1ファイルに入っています。フォーム表示、残工数計算、営業日計算、報告文生成、コピー、クリア、localStorage 保存を担当します。
- `server.mjs`: 依存なしの簡易ローカルサーバーです。`npm run dev` と `npm run preview` で使われます。
- `build.mjs`: `index.html` を `dist/index.html` にコピーするだけの簡易ビルドスクリプトです。
- `package.json`: npm scripts を定義しています。依存関係は空です。
- `README.md`: 起動方法の短い説明です。
- `dist/index.html`: `npm run build` で生成される配布用ファイルです。手編集せず、`index.html` を編集してから再ビルドしてください。

## 4. まだ未完成な部分、ダミーになっている部分

- AI連携は未実装です。現在の報告文生成は `index.html` 内の `generate()` によるルールベース生成です。
- 文章生成は固定テンプレートです。入力内容を自然言語として深く解釈したり、文脈に応じて言い換えたりはしません。
- 祝日判定はブラウザ内の簡易ロジックです。固定祝日、ハッピーマンデー、春分・秋分、振替休日、国民の休日を計算していますが、将来の特例祝日や法律変更には自動追従しません。
- データ保存はブラウザの `localStorage` のみです。ユーザーごとのクラウド保存、共有、履歴管理はありません。
- 入力例はプレースホルダのみです。サンプルデータ投入機能はありません。

## 5. 既知のバグや制約

- バックエンド、認証、データベースはありません。静的HTMLとして動かす設計です。
- 残作業の時刻は「何時」だけを整数で入力します。分単位の予定は扱いません。
- 工数計算は 9:00〜18:00 の範囲だけを対象にし、昼休みなどの休憩時間は差し引いていません。
- 祝日計算は日本向けです。他国の休日や会社独自休日には対応していません。
- クリップボードコピーは通常ブラウザでは動きますが、古いブラウザや一部のセキュリティ制限下ではフォールバック動作に依存します。
- `src` フォルダが空で残っている場合がありますが、現在のアプリ実装には使っていません。

## 6. Webチームへの推奨次ステップ

1. `index.html` 1ファイル実装を、React + Vite などの保守しやすい構成へ分割する。
2. 残作業行、営業日計算、祝日計算、メッセージ生成を個別モジュールに切り出し、単体テストを追加する。
3. 祝日データを公式CSVや運用チーム管理の設定ファイルから読み込めるようにする。
4. 昼休み、会社休日、勤務時間帯を設定可能にする。
5. OpenAI API などを使う場合は、APIキーをフロントエンドに置かず、Netlify Functions などの安全なサーバー側経由にする。
6. 生成文のトーン、宛先、緊急度を選べるUIを追加する。
7. アクセシビリティ、モバイル操作性、コピー成功・失敗時のフィードバックを本番品質に磨く。
