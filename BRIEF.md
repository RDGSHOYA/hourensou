# 報連相アシスタント

## 誰が使うのか

- 利用者は1名（本人専用）
- 日常的に業務の進捗報告、連絡、相談を行う会社員
- 上司や先輩への報告内容を整理したい人
- TeamsやSlackでのコミュニケーションを主に利用する人

## 解決する問題

利用者は報告・連絡・相談（報連相）を行う前に、現在の状況を整理することに時間がかかっている。

特に以下の課題を解決する。

- 現在の進捗状況を客観的に整理したい
- 進捗が遅れているかどうかを把握したい
- 遅れている場合に何を優先すべきか整理したい
- 困りごとやリスクを適切に言語化したい
- 上司や先輩が理解しやすい形で報告文を作成したい
- 報告後の追加質問を減らしたい

本アプリは入力された情報をもとに状況を整理し、TeamsやSlackへそのまま貼り付けられる実用的な報告文を生成する。

## ユーザーが入力する情報

入力フォームには以下の項目を用意する。

1. タスク名
2. 締切日
3. 現在の進捗率（0〜100%）
4. 完了した作業
5. 残っている作業
6. 困っていること
7. 想定工数
8. 実績工数
9. メモやチャット内容

各項目は複数行テキスト入力を基本とする。

## アプリが計算・表示するもの

アプリは入力内容をAIに渡し、以下を生成する。

### 状況整理

- 現在の進捗状況の要約
- 完了済み作業の整理
- 未完了作業の整理
- リスク要因の整理
- 困りごとの整理

### 報告文生成

AIは以下を満たす文章を生成する。

- TeamsやSlackへそのまま貼り付け可能
- 短く読みやすい
- 結論が先に来る
- 現状が分かる
- 進捗遅延があれば自然に表現する
- リスクがあれば自然に表現する
- 必要に応じて相談事項を含める
- 上司や先輩が追加質問しなくても理解できるレベルで整理する

### 出力エリア

生成結果はコピーしやすいテキスト形式で表示する。

機能例：

- ワンクリックコピー
- 再生成
- 入力内容保持

## スタイル(明るく、モダンで、編集デザイン風。
  ウォームライトグレー背景、白カード、ダークモード無し、サイドバー無し)

デザイン要件：

- 明るい印象
- モダンなSaaS風UI
- 編集デザイン風の余白設計
- ウォームライトグレー背景
- 白いカードUI
- 大きめの余白
- 読みやすいタイポグラフィ
- サイドバー無し
- ダークモード無し
- シングルページ構成

レイアウト：

- 上部にタイトル
- 中央に入力フォーム
- 下部に生成結果
- 最大幅を制限した読みやすいレイアウト
- モバイルとPCの両方に対応

## 「完成」の定義

以下を満たした時点で完成とする。

1. 利用者が必要情報を入力できる
2. AIが状況を整理できる
3. AIが報告文を生成できる
4. 出力内容をTeamsやSlackへそのまま貼り付けられる
5. 上司や先輩からの追加質問が減るレベルで情報が整理されている
6. 進捗遅延やリスクが自然に文章へ反映される
7. コピー操作が簡単にできる
8. スマホ・PCの両方で快適に利用できる

## 技術スタック — React + Vite + Tailwind、バックエンド無し、データベース無し、
  ログイン無し、Netlifyにデプロイできる静的サイト

必須構成：

- React
- Vite
- Tailwind CSS

制約：

- バックエンド無し
- データベース無し
- ユーザー認証無し
- ログイン機能無し
- Netlifyへデプロイ可能な静的サイト
- 単一ページアプリケーション（SPA）

推奨構成：

- App.tsx
- components/InputForm.tsx
- components/GeneratedReport.tsx
- hooks/useReportGenerator.ts
- lib/prompt.ts

MVPとして最短で実装できる構成を優先する。
